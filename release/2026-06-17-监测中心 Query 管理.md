# Release Notes - 2026-06-17

## 变更范围

本次 release 聚焦监测中心的信息架构和 Query 管理闭环：

- 监测中心
- 品牌 / 产品
- Query 管理
- 模型回复查看

## 监测中心

### 信息架构调整

新增和优化：

- 左侧菜单保留一级业务入口，减少二级菜单对主流程的干扰。
- “品牌 / 产品”作为独立一级入口，用于承载品牌、产品、竞品和知识资料。
- Query 管理从单条 Query 列表调整为 Query 集管理。
- 监测中心核心流程收敛为：
  - 维护品牌 / 产品基础资料
  - 创建 Query 集
  - 在 Query 集内维护 Query
  - 发起评测任务
  - 查看各模型回复和表现
  - 根据缺口准备内容

## 品牌 / 产品

### 页面结构

新增和优化：

- 新增“品牌 / 产品”页面组。
- 页面组内包含：
  - 品牌档案
  - 产品档案
  - 竞品配置
  - 品牌事实 / 知识资料
- 不在左侧菜单新增子菜单，页面组内部使用 Tab 切换。
- Tab 位置统一紧挨页面标题。
- 有独立 Tab 的页面统一标题区上下间距，避免不同页面之间视觉跳动。
- 品牌档案、内容数据等页面的标题、说明、Tab 间距保持一致。

### 品牌事实 / 知识资料

新增和优化：

- 将品牌知识资料纳入品牌 / 产品模块。
- 支持展示品牌事实、产品事实、服务政策等基础资料。
- 用于后续支撑 Query 推荐、模型回复校验和内容准备。

## Query 管理

### Query 集管理

新增和优化：

- Query 管理首页调整为 Query 集列表。
- 支持查看不同 Query 集的基础信息：
  - Query 集名称
  - 业务场景
  - 关联品牌
  - 关联产品
  - 状态
  - 负责人
  - 更新时间
  - 最近评测时间
- Query 集支持状态展示：
  - 草稿
  - 已冻结
  - 已归档
- 支持创建 Query 集。
- 创建 Query 集弹窗包含：
  - Query 集名称
  - 业务场景
  - 关联产品
  - 关联品牌
  - 负责人
  - 说明
- 创建成功后 Query 集自动出现在列表中。

### Query 集详情

新增和优化：

- Query 集详情页作为 Query 维护主工作台。
- 顶部展示 Query 集概览信息：
  - 状态
  - 版本
  - 业务场景
  - 关联品牌
  - 关联产品
  - 负责人
  - 最近评测
- 支持展示当前 Query 集统计：
  - Query 总数
  - 高优先级数量
  - 品牌缺席数量
  - 已推荐数量
- 支持手动添加 Query。
- 支持批量导入入口演示。
- 支持冻结 Query 集入口演示。
- 支持创建评测任务入口演示。

### Query 列表

新增和优化：

- 单条 Query 不再设置分类。
- Query 分类统一上移到 Query 集层面，用业务场景表达。
- 单条 Query 不再展示标签。
- Query 列表字段调整为：
  - Query
  - 优先级
  - 最近表现
  - 主要竞品
  - 操作
- 支持按 Query 内容搜索。
- 支持按优先级筛选。
- 支持分页，默认每页 10 条，可切换 20 / 50 条。
- “查看回复”和“准备内容”改为图标按钮。
- 操作区保留移除 Query。

### 最近表现

新增和优化：

- Query 列表的“最近表现”不再只展示单个汇总状态。
- 最近表现改为按模型展示简易状态。
- 每个模型展示：
  - 提及
  - 推荐
  - 缺席
- 当前 mock 覆盖模型：
  - 豆包
  - DeepSeek
  - Kimi
  - 通义千问
- 列表中只展示简易结果，完整回复仍在详情弹窗中查看。

### AI 推荐 Query

新增和优化：

- AI 推荐 Query 入口放到 Query 集详情页。
- 推荐弹窗支持按当前 Query 集上下文生成候选问题。
- 推荐配置包含：
  - 目标场景
  - 推荐方向
  - 当前品牌
  - 当前产品
  - 当前集合
- 支持选择多个候选 Query 后批量加入当前 Query 集。
- 加入后可继续在 Query 列表中维护优先级。
- 推荐 Query 不再带单条分类和标签。

## 模型回复查看

### 回复列表弹窗

新增和优化：

- Query 列表支持查看单条 Query 在不同批次评测任务中的模型回复。
- 查看回复以弹窗形式呈现，不打断当前 Query 集维护流程。
- 回复弹窗展示 Query 基础信息：
  - 所属 Query 集
  - 优先级
  - 最近表现
- 支持按模型筛选。
- 支持按评测批次筛选。
- 支持搜索回复内容或引用来源。
- 回复列表字段包括：
  - 批次
  - 模型
  - 判断
  - 回复内容
  - 引用来源
  - 任务
- 判断结果展示：
  - 是否提及
  - 是否推荐
- 支持从回复列表跳转查看对应评测任务。

## 产品规则沉淀

本次明确的产品规则：

- Query 集是业务管理单元。
- Query 集可以分类，使用业务场景表达。
- 单条 Query 只承载用户真实问题，不再承载分类和标签。
- AI 推荐 Query 应发生在 Query 集详情页，而不是全局列表页。
- Query 列表负责快速扫表现，完整模型回复放到查看回复弹窗中。
- 内容准备入口应从具体 Query 出发，便于承接缺口补内容。
